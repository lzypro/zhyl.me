As an underlying infrastructure company, SmartX needs to use architecture diagrams in a variety of contexts over time. These architecture diagrams are often used to illustrate the structure of an enterprise’s IT architecture. In general these diagrams are mainly created by pre-sales or system engineers, and used on documents or websites for solution explanations.

We organized the architecture diagram into two parts, software and hardware, with each component placed on the grid in a modular pattern. After a lot of experiments and disccusion with marketing department colleagues, we finally settled on a scheme of wireframe with text to represent the software and icons to represent the hardware. This design balances the technical community’s perception of the architecture diagram with the uniqueness of the SmartX brand.

Process

Goal

Modular, refined architecture for everyone.

Drawing architecture diagrams for technical solutions has always been a huge hassle — at first you need to find an ideal tool, then you need to search the web for icon stocks, and not to mention the need to develop fonts, colors and line styles. So our goal is to design an architecture diagram library that can be easily used by anyone.

Initial Goals

The typical architecture diagram in the industry is shown above, and the problems we want to solve include:

  1. No clear distiction between hardware and software.
  2. Not using professional, high-quality graphics.
  3. There is no grid to control the size and proportion of individual parts.
  4. Difficulty in reusing individual elements.
Scales & Grids

Expandable 8px grid for digital screen.

The primary use of architecture diagrams at SmartX is as illustrations to explain technical solutions in articles, and as images in presentation slides. At the same time we also need to take into acount of other sizes on digital screen. Therefore we need a dimension that prioritizes display in the width of article paragraphs, while having a grid that can be flexibly expanded.

Expandable Grid

We started out with a 12-column 8px grid, which can be expanded infinitely.

Colors & Icons

Designed to help identify.

To seperate the colors from the rest of SmartX’s products while maintaining some connection, I initially developed a set of color palettes. Then, in order to create a contrast with the structure of the software part, I tried to design icons by modeling with 3D software.

Shift to Minimalism

After much experimentation, we found that fully realistic 3D icons would be intrusive to the renderings of SmartX hardware products and confuse users. So we also tried to abstract the shape of the models and use Blender’s ColorRamp node to render the model as a flat image.

Variants

Thanks to Figma’s then newly introduced Variants feature, I used it to set different states for each icon. Users can easily switch between states to meet the needs of various scenarios.

Icons for Software

In fact I also designed the software icons at the same time, in order to distinguish different software. However we did not use them because of the technical staff’s habit and to save the space occupied by the images.
